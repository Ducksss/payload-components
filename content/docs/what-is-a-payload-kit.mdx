title: What is a Payload block kit?
description: A Payload block kit is an installable Payload CMS block plus the collection, renderer, type, and import-map wiring that makes it live in a Payload v3 project.
---

A **Payload block kit** is an installable unit of a Payload CMS site: a layout block's source _plus_ the four edits that make that block actually render in a Payload v3 + Next.js project.

Copying a block's files is the easy part. A pasted block is inert until it is registered, rendered, typed, and added to the admin import map. A kit packages the source and that wiring together, and `payload-kit add` applies all of it as one reviewable git diff.

## The four edits a block needs

A Payload layout block is not live just because its files exist. Four more edits have to land:

1. Register the block in `src/collections/Pages/index.ts`.
2. Map it in `src/blocks/RenderBlocks.tsx`.
3. Regenerate `src/payload-types.ts` (`payload generate:types`).
4. Regenerate the admin import map (`payload generate:importmap`).

A kit owns exactly those edits. See [payload-kit add vs shadcn add](/docs/shadcn-vs-payload-kit) for the artifact-by-artifact difference.

## What ships in a kit

A kit is more than source. It reaches the catalog only when four things ship together:

- **Source** — the block config and its frontend component.
- **Manifest** — registry metadata plus the files the install patches and regenerates.
- **Docs** — a contract page listing fields, installed files, and supported targets.
- **Installer coverage** — tests that prove the install and the idempotent re-install.

Half-wired blocks never reach the catalog.

## How a kit differs from…

- **A code snippet** — a snippet is text to paste. A kit is an install that also wires the block into your collection, renderer, types, and import map.
- **A shadcn component** — `shadcn add` copies files and stops. A Payload block needs four more edits to render; a kit wraps the same registry delivery with that wiring.
- **A Payload plugin** — a plugin is a runtime dependency you import and keep. A kit is plain source copied into your repo — no vendored framework, no lock-in. You own and can edit the result.
- **A starter template** — a template scaffolds a whole project once. A kit installs one block into a project you already run, and converges on a second install.

## Install one

```bash
npx payload-kit add hero-basic
```

The CLI checks your project against the support matrix (Payload v3 + Next.js 15/16, website-starter shape), copies the source, applies the two patches, runs the two regenerations, and records install state in `.payload-kit/state.json`.

- Read [Installation](/docs/installation) before running it.
- Browse the [Kit Catalog](/components) for the current kits.
