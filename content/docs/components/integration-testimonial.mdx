## Content model

`heading`, `subtext`, and `integrations` come from the shared `integrationFields` base. This variant
adds a customer `quote` with `author`, optional `role`, and optional `authorAvatar`.

<TypeTable
  type={{
    heading: {
      type: 'text',
      required: true,
      description: 'Short heading shown beside the logo grid.',
    },
    subtext: {
      type: 'textarea',
      description: 'Optional supporting paragraph under the heading.',
    },
    integrations: {
      type: 'array',
      required: true,
      description: 'Two to twelve integrations rendered as logo cards with a name and optional description.',
    },
    quote: { type: 'textarea', required: true, description: 'Customer quote shown beside the grid.' },
    author: { type: 'text', required: true, description: 'Name of the person quoted.' },
    role: { type: 'text', description: 'Optional role / company for the author.' },
    authorAvatar: { type: 'upload', description: 'Optional avatar image for the author.' },
  }}
/>

Each item in `integrations` carries:

<TypeTable
  type={{
    logo: { type: 'upload', required: true, description: 'Logo image (Media upload).' },
    name: { type: 'text', required: true, description: 'Accessible name / alt text for the logo.' },
    description: { type: 'textarea', description: 'Short blurb shown on the card.' },
    href: { type: 'text', description: 'Optional link (unused by this variant).' },
  }}
/>
