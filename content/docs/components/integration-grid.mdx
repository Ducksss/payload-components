## Content model

`heading`, `subtext`, and `integrations` come from the shared `integrationFields` base. The grid
renders each item's per-item `description` and, when set, a learn-more link from `href`.

<TypeTable
  type={{
    heading: {
      type: 'text',
      required: true,
      description: 'Short heading shown above the integration cards.',
    },
    subtext: {
      type: 'textarea',
      description: 'Optional supporting paragraph under the heading.',
    },
    integrations: {
      type: 'array',
      required: true,
      description: 'Two to twelve integrations, each an uploaded logo with a name, optional description, and optional link.',
    },
  }}
/>

Each item in `integrations` carries:

<TypeTable
  type={{
    logo: { type: 'upload', required: true, description: 'Logo image (Media upload).' },
    name: { type: 'text', required: true, description: 'Accessible name / alt text for the logo.' },
    description: { type: 'textarea', description: 'Short blurb shown on the card.' },
    href: { type: 'text', description: 'Optional learn-more link the card points to.' },
  }}
/>
