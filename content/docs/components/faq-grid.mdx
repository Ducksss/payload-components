## Content model

The first three fields come from the shared `faqFields` base; `items` is specific to this
variant. Unlike the accordion variants, every answer is shown at once — no interactive state.

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    description: { type: 'textarea', description: 'Optional intro copy.' },
    items: {
      type: 'array',
      required: true,
      description: 'One or more question/answer pairs rendered as a two-column card grid.',
    },
  }}
/>

Each item in `items` carries:

<TypeTable
  type={{
    question: { type: 'text', required: true, description: 'The card heading.' },
    answer: { type: 'textarea', required: true, description: 'The card body copy.' },
  }}
/>
