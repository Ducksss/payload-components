## Content model

The first three fields come from the shared `featureFields` base; `items` and `links` are specific
to this variant.

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    description: { type: 'textarea', description: 'Optional intro copy beside the list.' },
    items: {
      type: 'array',
      required: true,
      description: 'Two to six feature rows, each with a required title and description.',
    },
    links: { type: 'array', description: 'One or two optional CTA links.' },
  }}
/>

Each item in `items` carries:

<TypeTable
  type={{
    title: { type: 'text', required: true, description: 'Feature row heading.' },
    description: { type: 'textarea', required: true, description: 'Feature row body copy.' },
  }}
/>
