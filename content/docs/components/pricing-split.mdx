## Content model

The first three fields come from the shared `pricingFields` base; `plans` is built from the
shared `planFields`. This variant expects exactly two plans: the first renders as the compact
entry column on the left, the second as the expanded panel on the right (mark it `featured`
for the emerald "Popular" badge).

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    description: { type: 'textarea', description: 'Optional intro copy.' },
    plans: {
      type: 'array',
      required: true,
      description: 'Exactly two plans: [0] entry plan, [1] expanded featured plan.',
    },
  }}
/>

Each item in `plans` carries:

<TypeTable
  type={{
    name: { type: 'text', required: true, description: 'Plan name, e.g. "Pro".' },
    price: { type: 'text', required: true, description: 'Headline price, e.g. "$19".' },
    period: { type: 'text', description: 'Billing period, e.g. "/ mo".' },
    description: { type: 'text', description: 'Short plan blurb.' },
    featured: { type: 'checkbox', description: 'Shows the emerald "Popular" badge on the right plan.' },
    features: { type: 'array', required: true, description: 'Included features, each a single line of text.' },
    links: { type: 'array', description: 'One CTA link for the plan.' },
  }}
/>
