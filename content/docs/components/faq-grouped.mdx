## Content model

The first three fields come from the shared `faqFields` base; `groups` is specific to this
variant. Each group's `icon` is a `select` over the `faqIcons` allowlist — Payload stores the
name and the frontend looks up the matching lucide mark at render.

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    description: { type: 'textarea', description: 'Optional intro copy.' },
    groups: {
      type: 'array',
      required: true,
      description: 'One or more titled, icon-tagged category groups.',
    },
  }}
/>

Each group in `groups` carries:

<TypeTable
  type={{
    icon: { type: 'select', description: 'Icon name from the shared faqIcons allowlist.' },
    title: { type: 'text', required: true, description: 'The category heading.' },
    items: {
      type: 'array',
      required: true,
      description: 'The question/answer rows for this category, rendered as an accordion.',
    },
  }}
/>
