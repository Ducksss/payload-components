## Content model

`heading`, `subtext`, and `integrations` come from the shared `integrationFields` base. Each row
renders the per-item `description` and, when set, links its add action to `href`.

<TypeTable
  type={{
    heading: {
      type: 'text',
      required: true,
      description: 'Short heading shown above the list.',
    },
    subtext: {
      type: 'textarea',
      description: 'Optional supporting paragraph under the heading.',
    },
    integrations: {
      type: 'array',
      required: true,
      description: 'Two to twelve integration rows, each an uploaded logo with a name, optional description, and optional link.',
    },
  }}
/>

Each item in `integrations` carries:

<TypeTable
  type={{
    logo: { type: 'upload', required: true, description: 'Logo image (Media upload).' },
    name: { type: 'text', required: true, description: 'Accessible name / alt text for the logo.' },
    description: { type: 'textarea', description: 'Short blurb shown on the row.' },
    href: { type: 'text', description: 'Optional link the add action points to.' },
  }}
/>
