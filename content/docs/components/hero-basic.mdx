  `hero-basic` is the **golden alpha component** — the reference install the whole wrapper
  workflow is validated against.
</Callout>

```bash
npx payload-components add hero-basic
```

## What it installs

Three source files land in your project:

<Files>
  <Folder name="src/blocks/shared">
    <File name="heroFields.ts" />
  </Folder>
  <Folder name="src/blocks/HeroBasic" defaultOpen>
    <File name="config.ts" />
    <File name="Component.tsx" />
  </Folder>
</Files>

…and two fragments are wired automatically:

- a render-block registration for `heroBasic`
- a `Pages` layout registration for `HeroBasic`

<Callout type="info">
  `heroFields.ts` is the shared field core for the Hero family — every hero variant
  composes it. Editing it updates each installed hero block at once, and re-running an
  install never overwrites a copy you have changed.
</Callout>

## Acceptance criteria

<Callout type="success">
  The component is valid when it installs into a supported target project, registers once
  in `RenderBlocks`, registers once in the `Pages` layout field, runs post-install
  scripts, and treats a second install as idempotent.
</Callout>
