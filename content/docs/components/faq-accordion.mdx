## Content model

The first three fields come from the shared `faqFields` base; `items` and `links` are specific
to this variant.

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    description: { type: 'textarea', description: 'Optional intro copy.' },
    items: {
      type: 'array',
      required: true,
      description: 'One or more question/answer pairs rendered as accordion rows.',
    },
    links: { type: 'array', description: 'One optional CTA link below the accordion.' },
  }}
/>

Each item in `items` carries:

<TypeTable
  type={{
    question: { type: 'text', required: true, description: 'The accordion trigger text.' },
    answer: { type: 'textarea', required: true, description: 'The answer revealed when expanded.' },
  }}
/>
