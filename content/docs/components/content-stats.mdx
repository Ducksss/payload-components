## Content model

The `eyebrow`, `title`, and `paragraphs` fields come from the shared Content family base
(`src/blocks/shared/contentFields.ts`); the `features` grid and `stats` list are specific to this
variant. Feature icons are chosen from the shared allowlist in `src/blocks/shared/contentIcons.ts`.

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Optional small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    paragraphs: {
      type: 'array',
      description: 'Intro copy — one or more short paragraphs (min 1, max 4).',
    },
    features: {
      type: 'array',
      description: 'Two to six icon features rendered in a responsive grid.',
    },
    stats: {
      type: 'array',
      description: 'A list of headline stats (value + label), max 8.',
    },
  }}
/>

Each item in `features` carries `icon` (a Lucide name: zap · cpu · lock · sparkles · gauge · shield),
`title`, and `description`. Each item in `stats` carries:

<TypeTable
  type={{
    value: { type: 'text', required: true, description: 'The headline figure, e.g. “90+”.' },
    label: { type: 'text', required: true, description: 'What the figure measures.' },
  }}
/>
