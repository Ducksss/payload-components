<Callout type="info">
  `feature-grid-basic` proves the second-kit path and repeatable content fields —
  installable before or after `hero-basic`.
</Callout>

```bash
npx payload-kit add feature-grid-basic
```

## What it installs

<Files>
  <Folder name="src/blocks/FeatureGridBasic" defaultOpen>
    <File name="config.ts" />
    <File name="Component.tsx" />
  </Folder>
</Files>

…and two fragments are wired automatically:

- a render-block registration for `featureGridBasic`
- a `Pages` layout registration for `FeatureGridBasic`

## Content model

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    description: { type: 'textarea', description: 'Optional intro copy.' },
    items: {
      type: 'array',
      required: true,
      description: 'Three to six feature cards, each with a required title and description.',
    },
    links: { type: 'array', description: 'One optional CTA link.' },
  }}
/>

Each item in `items` carries:

<TypeTable
  type={{
    title: { type: 'text', required: true, description: 'Feature card heading.' },
    description: { type: 'textarea', required: true, description: 'Feature card body copy.' },
  }}
/>

## Acceptance criteria

<Callout type="success">
  The kit is valid when it can install before or after `hero-basic`, dedupes all
  fragments, and keeps each feature item readable on mobile and desktop.
</Callout>
