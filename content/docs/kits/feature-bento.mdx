## Content model

The first three fields come from the shared `featureFields` base; `items` and `links` are specific
to this variant.

<TypeTable
  type={{
    eyebrow: { type: 'text', description: 'Small label shown above the title.' },
    title: { type: 'text', required: true, description: 'The section heading.' },
    description: { type: 'textarea', description: 'Optional intro copy.' },
    items: {
      type: 'array',
      required: true,
      description: 'Three to six cells; the first leads the grid as the featured cell.',
    },
    links: { type: 'array', description: 'One optional CTA link.' },
  }}
/>

Each item in `items` carries:

<TypeTable
  type={{
    title: { type: 'text', required: true, description: 'Cell heading.' },
    description: { type: 'textarea', required: true, description: 'Cell body copy.' },
  }}
/>
