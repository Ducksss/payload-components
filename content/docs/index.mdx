Payload Kits is an open-source, community-first registry and CLI that installs Payload CMS block kits into your **Payload v3 + Next.js** project — **wired, not pasted**.

<Callout type="info" title="What makes a kit more than a copy-paste block">
  A plain `shadcn add` only copies files. `payload-kit add` also does the Payload
  wiring — it registers the block in your collection, maps the renderer, and
  regenerates types plus the admin import map — then lands the whole thing as a
  reviewable git diff.
</Callout>

The v2 app is intentionally not a Payload CMS site. It is a Fumadocs app with MDX documentation, static registry assets, and package tooling for validating installs into external Payload projects.

## Install your first kit

```bash
npx payload-kit add hero-basic
```

The CLI verifies project shape, installs the registry item, applies Payload layout fragments, runs post-install tasks, and records install state.

<Callout type="warn">
  Run this inside a supported **Payload v3 + Next.js App Router** project — not
  against this docs repo. [Installation](/docs/installation) lists the requirements
  the installer checks before it touches anything.
</Callout>
