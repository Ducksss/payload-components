Payload Kits ships installable Payload CMS v3 kits through a shadcn-compatible registry. shadcn-native kits are the public front door; `payload-kit` is reserved for kits that need Payload-aware project mutation.

## What These Docs Cover

- how `npx shadcn add @payload-kits/post-card` works
- what `payload-kit add` changes when a kit needs Payload wiring
- which kits can be pure shadcn Directory installs
- which files a kit owns
- how schema registration, render wiring, generated types, and import maps are handled
- where to preview the actual shipped components before installation

## Current Alpha Surface

## The Installation Contract

Payload Kits has two install contracts. shadcn-native kits are complete after file and dependency delivery. payload-kit-required kits are complete only after block source, Payload config, renderer registration, generated types, and import-map updates are in a consistent state.

```bash
npx shadcn add @payload-kits/post-card
npx payload-kit add hero-basic
```
