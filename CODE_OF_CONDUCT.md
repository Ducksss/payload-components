# Code of Conduct

Payload Kits should be a useful, low-friction place for Payload CMS developers
to compare implementation ideas, report issues, and improve installable kits.

## Standards

Expected behavior:

- Be direct, specific, and respectful.
- Assume technical disagreement is normal and keep it focused on the work.
- Give enough reproduction detail for bugs and enough context for proposals.
- Respect maintainers' decisions about project scope and priorities.

Unacceptable behavior:

- Harassment, threats, personal attacks, or discriminatory language.
- Repeated off-topic comments after a maintainer has redirected the thread.
- Publishing private information without permission.
- Bad-faith reports, spam, or attempts to disrupt the project.

## Enforcement

Maintainers may edit, hide, or remove comments; lock discussions; close issues;
or block users when needed to keep the project usable for contributors and
early adopters.

If you need to report conduct concerns privately, contact the maintainer listed
in the README.
