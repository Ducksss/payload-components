# Payload Kits 4-Week Launch Playbook

## Owner Metric

Use one top-line metric for the first push:

- `qualified waitlist signups`
- `design-partner conversations`

Everything else should support that learning loop.

## Week 1

- Tighten GitHub as the public proof surface.
- Point README traffic to the early-access issue template.
- Share the first guide from `/resources`.
- Start tagging all waitlist links with `source` and `utm_*`.

## Week 2

- Publish 1-2 community posts with useful teardown angles.
- Reach out to 10-15 agencies or freelancers already shipping Payload websites.
- Watch which sources produce design-partner intent instead of generic curiosity.

## Week 3

- Publish the second and third guides.
- Repost the strongest community angle in a second relevant channel.
- Refine hero and CTA copy based on the sources that convert.

## Week 4

- Follow up with the most relevant early-access and design-partner leads.
- Capture install objections, missing kit categories, and repo-shape blockers.
- Decide whether the next step is more content, more GitHub proof, or a small launch moment.

## Source Conventions

Use lowercase kebab-case values for `source`.

Recommended starting values:

- `github-readme`
- `github-issue-template`
- `payload-discord-post-01`
- `payload-discussions-post-01`
- `resource-payload-cms-blocks`
- `resource-payload-nextjs-template`
- `resource-shadcn-registry-payload`
- `direct-outreach-agency`
- `direct-outreach-freelancer`

## Link Shapes

Examples to reuse:

```txt
/?source=github-readme&utm_source=github&utm_medium=readme&utm_campaign=early-access#early-access
/?source=payload-discord-post-01&utm_source=payload-discord&utm_medium=community&utm_campaign=launch-education#early-access
/?source=direct-outreach-agency&utm_source=outreach&utm_medium=dm&utm_campaign=design-partner&intent=design-partner#early-access
/resources/payload-cms-blocks
/resources/shadcn-registry-for-payload
```

## Success Signals

- GitHub traffic produces early-access requests without paid traffic.
- Community posts generate replies from people already operating Payload projects.
- Resource guides begin earning search impressions for narrow Payload queries.
- Direct outreach turns into 5-10 real conversations and a few install attempts.
