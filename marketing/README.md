# Payload Kits Launch Assets

These files turn the early channel plan into execution assets you can actually use week to week.

## Files

- [`launch-playbook.md`](./launch-playbook.md): 4-week sequence, owner metric, and UTM/source conventions
- [`community-posts.md`](./community-posts.md): draft post angles and CTAs for Payload-heavy communities
- [`outreach.md`](./outreach.md): qualification rubric plus founder-led outreach templates

## Working Defaults

- Goal: qualified waitlist signups and design-partner conversations
- Audience: Payload v3 + Next.js agencies, freelancers, and consultants
- Budget: mostly organic
- Core channel mix: GitHub, Payload communities, owned guides, and direct outreach

The homepage waitlist flow now accepts `source`, `intent`, `role`, and `utm_*` context, so use the conventions in the playbook when sharing links.
