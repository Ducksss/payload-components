# Founder-Led Outreach

## Who To Contact First

Prioritize people who already have proof that they ship Payload work:

- agencies building marketing sites on Payload
- freelancers and consultants who start from the official website-style template
- educators or creators already teaching Payload and Next.js workflows
- starter-kit or template authors with adjacent distribution

## Qualification Signals

Green flags:

- ships more than one Payload site per quarter
- already has repeated block patterns
- cares about preview behavior and editor confidence
- is comfortable testing alpha installs in a real repo

Lower priority for now:

- teams outside Payload v3 + Next.js App Router
- people mainly interested in broad startup tooling
- anyone asking for a marketplace before the install path is proven

## Short DM Template

```txt
Hey {name} - I am building Payload Kits, a Payload-native install flow for reusable website blocks.

The focus is narrow on purpose: Payload v3 + Next.js repos, especially agencies and freelancers who keep rebuilding hero, pricing, FAQ, CTA, and form sections.

I am looking for a few early design partners who can pressure-test the install flow in real projects. If that sounds relevant, I would love to show you the current proof and hear where block reuse tends to break in your repos.
```

## Conversation Goals

In each call or async thread, learn:

- which block categories they rebuild repeatedly
- where installs usually break in mildly customized repos
- whether preview, types, or import-map drift causes real pain
- what would make a public free kit trustworthy enough to try
